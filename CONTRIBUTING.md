# 贡献指南

感谢你愿意改进《AI协作开发前期规范》。

## 可以贡献什么

- 改进文档
- 新增模板
- 新增检查表
- 新增示例项目
- 改进中英文翻译
- 提出流程问题和改进建议

## 贡献原则

1. 内容必须清晰、可执行。
2. 模板必须包含用途、字段说明和示例。
3. 示例项目必须展示完整前期流程。
4. 中英文版本应尽量保持结构一致。
5. 平台专用规则应放入对应文档或示例，不应污染通用规则。

## 新增示例项目要求

新增示例项目时，请至少包含：

- `README.md`
- `README.en.md`
- `preflight.md`
- `preflight.en.md`
- `development-spec.md`
- `development-spec.en.md`

## 新增模板要求

新增模板时，请包含：

- 用途
- 适用场景
- 字段说明
- 示例填写
- 常见错误

## Pull Request 检查

提交前请确认：

- Markdown 格式正常。
- 文件名和目录结构符合现有规范。
- 中文和英文文件尽量同步。
- 没有加入与主题无关的内容。
