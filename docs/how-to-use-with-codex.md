# 如何配合 Codex 使用

## 目标

让 Codex 在写代码前先读取本项目规范，完成需求澄清和开发规格设计。

## 推荐给 Codex 的提示词

```text
请先读取本仓库的 skill.md、docs/workflow.md 和 templates/core/development-spec.md。之后按 AI 协作开发前期规范执行：先复述项目构想，再分轮反问关键问题，输出决策锁定表、风险清单和开发规格。在规格未确认前，不要进入正式实现。
```

## 用于已有项目

```text
请读取当前仓库代码和本规范。先不要改代码。请先输出：
1. 当前项目理解
2. 项目类型
3. 已实现功能
4. 疑似问题
5. 需要用户确认的问题
6. 后续修改计划
```

## 用于新项目

```text
请按 skill.md 中的流程，从需求澄清开始，逐步生成开发规格，然后再实现项目。
```

## 最佳实践

- 先让 Codex 生成规格，不要马上改代码。
- 每次修改前先确认影响范围。
- 每次提交后要求 Codex 输出变更摘要和测试方式。
