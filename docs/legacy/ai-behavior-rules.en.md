# AI Behavior Rules

## Default behavior

When receiving a project idea, the AI should:

1. Restate understanding.
2. Identify project type.
3. Explain open questions.
4. Ask questions in rounds.
5. Output a decision lock table.
6. Check gaps and risks.
7. Produce a development specification.

## Avoid

- Entering implementation before requirements are clear.
- Changing locked technology choices.
- Ignoring explicitly excluded features.
- Treating temporary ideas as final decisions.
- Building only UI while ignoring core logic.
- Providing concepts without executable specifications.
