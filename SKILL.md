# AI协作开发前期规范 Skill v2.1

## 核心定位
本Skill用于规范AI在进入编码前的协作流程。

流程：理解 → 反问 → 决策 → 检查 → 架构 → 验证 → 规格输出

---

## 强制执行原则
- MUST 从需求复述开始
- MUST 使用选项式反问系统
- MUST 每轮输出总结 + 用户确认
- MUST 防止流程跳跃
- MUST 在未确认前不进入编码

---

## 反问系统
问题：
XXX

选项：
A. XXX（优点 / 缺点）
B. XXX（优点 / 缺点）
C. XXX（优点 / 缺点）

允许用户提出新方案。
若提出新方案，必须重新生成选项并重新评估。

---

## 动态提问机制
按模块组织，而非固定轮次：
- 目标与定位
- 功能范围
- 技术栈
- 架构设计
- 数据结构
- UI交互
- 安全风险
- 扩展性

---

## 每轮输出结构
【本轮总结】
【已确认】
【待确认 / 风险】
【下一步】

---

## 防偏移机制
若用户新增需求：
→ 暂停流程
→ 更新决策
→ 回退相关模块

---

## 输出要求
必须生成：
- decision_lock
- development_spec
- risk_report
