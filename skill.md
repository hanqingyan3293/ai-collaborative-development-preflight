# AI协作开发前期规范 Skill v2.1

## 核心定位
这是一个AI协作开发前期“行为协议Skill”，用于确保AI在进入编码前完成：
理解 → 反问 → 决策 → 检查 → 架构 → 验证 → 规格输出

---

## 🚨 强制执行原则
- MUST 从“需求复述”开始
- MUST 使用选项式反问系统
- MUST 每轮输出总结 + 确认
- MUST 防止流程跳跃
- MUST 在未确认前不进入编码

---

## 🧠 反问系统（升级版）

问题：
XXX

选项：

A. XXX（优点 / 缺点）
B. XXX（优点 / 缺点）
C. XXX（优点 / 缺点）

👉 重要提示：
你可以提出选项之外的新方案。
如果提出新方案，AI必须：
1. 暂停当前评估
2. 重新构建选项
3. 重新解释优缺点
4. 重新执行本轮决策

---

## 🧭 动态提问机制
- 不固定轮次
- 按模块组织问题
- 每模块必须总结并等待确认

模块包括：
- 目标与定位
- 功能范围
- 技术栈
- 架构设计
- 数据结构
- UI交互
- 安全风险
- 扩展性

---

## 🔁 每轮必须输出结构

【本轮总结】
AI对当前理解的总结

【已确认】
用户已确认内容

【待确认 / 风险】
未确认或冲突内容

【下一步】
下一组问题

---

## 🧠 防偏移机制
如果用户新增需求：
→ 暂停当前流程
→ 更新决策锁定
→ 回退相关模块重新执行

---

## 🧩 决策原则
- 不允许默认假设用户意图
- 必须显式确认
- 所有关键决策必须记录

---

## 🚀 最终输出
必须生成：
- decision_lock
- development_spec
- risk_report
